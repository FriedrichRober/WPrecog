  [33X[0;0YThis  chapter serves as an introduction and showcases some highlights of the
  package [5XWPrecog[105X.[133X
  
  
  [1X1.1 [33X[0;0YOverview[133X[101X
  
  [33X[0;0YThe  package  [5XWPrecog[105X  ([13XW[113Xreath  [13XP[113Xroduct  [13XRecog[113Xnition).  provides recognition
  methods  for  groups  abstractly isomorphic to certain (subgroups of) wreath
  products.[133X
  
  [33X[0;0YAssume  we  are  given as input a black box group [22XG[122X which is isomorphic to a
  (subgroup  of)  wreath  product.  In  certain cases, we can compute a wreath
  product  decomposition  of  the  input  group, that is an embedding from the
  group   into   a   natural   wreath   product  where  elements  are  encoded
  component-wise.  For example, [22XG[122X can be a matrix group of Aschbacher class C2
  or  a  primitive  group  of  type  PA.  The  main  command  is  provided via
  [10XWreathProductDecomposition[110X.[133X
  
  [33X[0;0YFor  instance, the computation of a wreath product decomposition of [22XPSL(3,3)
  ≀  E(4)[122X  represented  as a primitive group of type PA acting on [22X13^4 = 28561[122X
  points  takes less than half a second. This yields an effective mapping into
  an imprimitive wreath product acting on [22X13 ⋅ 4 = 52[122X points.[133X
  
  
  [1X1.2 [33X[0;0YPrimitive Groups of type PA[133X[101X
  
  [33X[0;0YWe  provide  an  example computation of a wreath product decomposition for a
  primitive permutation group [22XG[122X of type PA isomorphic to [22XPSL(3,3) ≀ E(4)[122X. As a
  User,  we  simply  need  to load the package and everything works out of the
  box.  However,  in  order  to  create  this  example, we additionally load a
  utility  file  that  helps  us  to  create pseudo-random generators. We also
  disable the additional display of information provided by the recog package.[133X
  
  [4X[32X  Example  [32X[104X
    [4X[28X# Load package and utility file for constructing pseudo-random generators, etc.[128X[104X
    [4X[25Xgap>[125X [27XLoadPackage("WPrecog", false);;[127X[104X
    [4X[25Xgap>[125X [27XReadPackage("WPrecog", "examples/Utils.gi");;[127X[104X
    [4X[25Xgap>[125X [27XSetInfoLevel(InfoRecog, 0);[127X[104X
  [4X[32X[104X
  
  [33X[0;0YNext,  we construct the components of the wreath product. The base component
  [22XK  = PSL(3,3)[122X acts on [22Xn = 13[122X points and the top component [22XE(4)[122X acts on [22Xm = 4[122X
  points.[133X
  
  [4X[32X  Example  [32X[104X
    [4X[28X# Construct components[128X[104X
    [4X[25Xgap>[125X [27XK := PrimitiveGroup(13, 7); # PSL(3,3)[127X[104X

    [4X[28X13[128X[104X
    [4X[25Xgap>[125X [27XLength(GeneratorsOfGroup(K));[127X[104X
    [4X[28X2[128X[104X
    [4X[25Xgap>[125X [27XH := TransitiveGroup(4, 2);[127X[104X
    [4X[28XE(4) = 2[x]2[128X[104X
    [4X[25Xgap>[125X [27Xm := NrMovedPoints(H);[127X[104X
    [4X[28X4[128X[104X
    [4X[25Xgap>[125X [27XLength(GeneratorsOfGroup(H));[127X[104X
    [4X[28X2[128X[104X

  [33X[0;0YNow,  we  construct  the  wreath  product  [22XW = K ≀ H[122X. The native [5XGAP[105X command
  [10XWreathProductProductAction(K,  H)[110X creates a wreath product in product action
  acting on [22X13^4 = 28561[122X points. Since [5XGAP[105X knows that we created [22XW[122X as a wreath
  product,  we  can  directly access the components of elements. Therefore, we
  later need to disguise the group on the computer so [5XGAP[105X does not have direct
  access  to this information. Moreover, the constructed generators are highly
  structured. The first [22X8[122X generators are the generators of the base group, and
  the last [22X2[122X generators are the generators of the top group.[133X

    [4X[25Xgap>[125X [27XListWreathProductElement(W, W.6);[127X[104X

  [33X[0;0YNow, we construct the black box group [22XG[122X isomorphic to [22XW = K ≀ H[122X. For this we
  use  utility  functions  provided by "examples/Utils.gi" in this package. In
  order  to  disguise  the  group  on  the  computer, we use a utility command
  [10XRandomGroup[110X  to  generate  pseudo-random  generators.  Afterwards,  we use a
  pseudo-random    permutation    generated    by    the    utility    command
  [10XQuickRandomPermutedList[110X to conjugate these generators. Therefore, [5XGAP[105X has no
  longer  access  to  the components of wreath product elements, even provided
  that we know the wreath product [22XW[122X.[133X
